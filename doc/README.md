# 🍷 BYOB 專案開發文檔（更新版）

## 📋 專案概述

BYOB (Bring Your Own Bottle) 是一個餐廳資訊平台，讓消費者可以找到支援自帶酒水的餐廳。本專案使用 WordPress + WooCommerce 作為後端，並整合自定義的餐廳業者會員系統。

### 🎯 專案目標

* 建立完整的餐廳業者會員系統
* 提供餐廳資料管理功能
* 實現自動化餐廳資料收集
* 建立專業的餐廳邀約追蹤機制
* 建立 SEO 優化與內容行銷系統
* 整合 SendGrid Email 發送系統
* 建立餐廳業者註冊通知系統
* 建立專業內容行銷策略
* 建立短影音行銷系統
* **啟動「雙軌制」策略：顧客端爆料 + 餐廳端先收錄** ⭐ 新增

## 📊 專案策略更新 — 「雙軌制」

### 問題背景

* 餐廳端回覆慢、摩擦高；Email 行銷開信率與 CTA 點擊率偏低。
* 顧客端具有天然動機（分享/索取名單），能形成社群內容與口碑循環。

### 新策略：雙軌制

* **顧客軌**：

  * 啟用「爆料 BYOB 餐廳」表單（簡單 3–5 問）
  * 設定自動回覆（感謝信 + 抽獎通知）
  * 誘因設計：每月抽獎（餐酒券 / 酒杯 / 禮券）、排行榜機制
  * 資料串接 Google Sheet/Airtable，標記 `source=customer_tip`
  * 每週公布新增名單，強化參與感

* **餐廳軌**：

  * 先收錄餐廳基本資料（以爬取資料為基礎）
  * 自動生成簡版餐廳卡片，標記 `source=auto_listed`
  * 通知餐廳「已免費上架」，引導一鍵補資料
  * 追蹤：補資料轉化率、通知開啟率

### 預期效益

* 快速累積餐廳名單（顧客爆料 + 爬取先收錄）
* 提高顧客參與度（抽獎、排行榜、社群互動）
* 降低餐廳端進入門檻（無需主動填表即可上架）
* 增加餐廳轉化誘因（FOMO 效應：別人已在平台上）
