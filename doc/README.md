# 🍷 BYOB 專案文檔

## 📋 專案概述

BYOB (Bring Your Own Bottle) 是一個自帶酒水餐廳推薦平台，目前運營兩個獨立專案：

### 🇹🇼 台北 BYOB（主要專案）
- **定位**：台北自帶酒水餐廳推薦平台
- **階段**：核心系統完成，推廣與酒商合作階段
- **策略**：抽獎激勵 + 多平台推廣

### 🇺🇸 費城 BYOB（新專案）
- **定位**：Yelp 的 BYOB 專業補充平台
- **階段**：專案規劃完成，準備啟動資料收集
- **策略**：Yelp 資料收集 + Reddit 社群驗證 + 榮譽系統驅動

  * 更新 `requirements.txt` 支援 API 方案（移除 Selenium、BeautifulSoup）

* ✅ Yelp API 申請流程研究
  * 研究 Yelp Fusion API 申請步驟和需求
  * 準備開發者帳號註冊資訊
  * 遇到 hCaptcha 重複驗證問題，提供多種解決方案

**🔧 技術決策分析**
- **為什麼選擇 Yelp Fusion API？**
  - ✅ 穩定性：官方 API，不會被阻擋或封鎖
  - ✅ 效率：結構化 JSON 資料，易於處理和去重
  - ✅ 成本：免費額度足夠需求（500 次/天 vs 8 次搜尋）
  - ✅ 品質：直接從 Yelp 資料庫獲取，資料準確可靠
  - ✅ 合規性：符合 Yelp 使用條款，避免法律風險

**🚨 當前阻塞問題**
- **hCaptcha 重複驗證問題**：完成驗證後重複跳出，無法完成 Yelp 帳號註冊
- **影響**：無法申請 API Key，阻礙爬蟲開發進度
- **解決方案**：清除瀏覽器資料、無痕模式、更換瀏覽器、VPN 更換 IP
- **明日優先處理**：按順序嘗試各種解決方案

## 🚨 當前技術挑戰與解決方案

### Yelp 爬蟲技術演進

**第一版嘗試（直接爬取）：**
- 使用 requests + BeautifulSoup
- 遇到 403 Forbidden 錯誤
- 原因：Yelp 反爬蟲機制（User-Agent 偵測、請求頻率限制）

**第二版嘗試（手動清單）：**
- 建立 15 家知名 BYOB 餐廳清單
- 問題：數量不足、不符合自動化需求
- 決定：刪除手動方案，專注自動化

**第三版方案（Yelp Fusion API）：**
- 使用官方 API，穩定可靠
- 每日 500 次免費請求，足夠需求
- 結構化 JSON 資料，易於處理
- 符合使用條款，避免法律風險

### 技術決策學習

**從失敗中學習：**
1. **直接爬取風險高**：現代網站都有強力的反爬蟲機制
2. **手動方案不可擴展**：不符合大規模資料收集需求
3. **API 方案最穩定**：官方支援，長期可持續

**最佳實踐：**
- 優先考慮官方 API
- 準備多種備用方案
- 重視合規性和可持續性
- 快速迭代和調整策略
