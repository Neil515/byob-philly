# 🍷 BYOB 專案開發文檔

## 📋 專案概述

BYOB (Bring Your Own Bottle) 是一個餐廳資訊平台，讓消費者可以找到支援自帶酒水的餐廳。本專案使用 WordPress + WooCommerce 作為後端，並整合自定義的餐廳業者會員系統。

### 🎯 專案目標
- 建立完整的餐廳業者會員系統
- 提供餐廳資料管理功能
- 實現LOGO和圖片上傳管理
- 建立菜單和照片管理系統
- 提供管理員備用介入管道
- 建立 SEO 優化與餐廳邀約發散策略
- 實現自動化餐廳資料收集
- 建立社群媒體管理與邀約系統
- 開發餐廳Email搜尋系統
- 建立獨立SEO頁面與混合策略
- 整合SendGrid Email發送系統
- 建立專業的餐廳邀約追蹤機制

---

## 🏗️ 技術架構

### 核心技術
- **後端**: WordPress 6.4.2 + WooCommerce 8.5.2
- **主題**: Flatsome Child Theme
- **外掛**: Advanced Custom Fields (ACF) Pro + Rank Math SEO
- **API**: Google Places API + Google Analytics + Google Search Console + SendGrid API
- **爬蟲**: Python + Google Places API + Excel 輸出 + 網格搜尋系統
- **社群媒體**: Facebook 粉絲專頁 + Instagram + Line 官方帳號
- **Email搜尋**: Python + Selenium + BeautifulSoup + requests
- **Email發送**: SendGrid + Domain Authentication + Python SDK
- **SEO優化**: Rank Math SEO + 圖片優化 + 外部連結策略
- **自定義功能**: 餐廳業者會員系統、餐廳資料管理、LOGO上傳、SEO 監控、自動化資料收集、社群媒體管理、資料同步修復、分頁導航、懶載入功能、Email搜尋系統、獨立SEO頁面、SendGrid Email發送、網格搜尋系統、多中心點搜尋、搜尋記錄機制、斷點續傳功能等

## 🚀 最新開發進度

## 📅 2025年9月18日 - Google Places API爬蟲程式60筆限制突破 ⭐ 重大技術突破

### 🎯 今日主要目標
突破Google Places API的60筆限制，建立網格搜尋系統，大幅提升餐廳資料收集能力。

### ✅ 已完成工作

#### 1. Google Places API限制分析 ⭐ 技術分析
**問題發現：**
- **API硬性限制**：Google Places API Text Search最多只能返回60筆結果（3頁×20筆）
- **無法突破**：這是Google的API限制，無法透過增加MAX_RESULTS來解決
- **搜尋範圍限制**：單一搜尋關鍵字無法獲得更多資料

**解決方案設計：**
- **網格搜尋策略**：將搜尋區域劃分為多個小網格
- **多中心點搜尋**：使用不同區域作為搜尋中心點
- **搜尋記錄機制**：避免重複搜尋相同區域

#### 2. 網格搜尋系統實作 ⭐ 核心功能
**技術實作：**
- **網格點生成**：每個中心點生成11×11=121個網格點
- **搜尋半徑設定**：0.03度（約3公里）的搜尋範圍
- **位置參數**：每個網格點使用location參數提升搜尋精確度
- **去重複機制**：使用place_id避免同一餐廳被重複收集

**系統架構：**
- **多中心點搜尋**：12個台北主要區域作為搜尋中心點
- **搜尋記錄檔案**：search_log.json記錄已搜尋的網格點
- **斷點續傳**：支援中斷後從上次位置繼續搜尋
- **進度保存**：每10個網格點自動保存進度

#### 3. 多中心點搜尋系統建立 ⭐ 系統架構
**中心點設定：**
- **12個台北區域**：信義區、中山區、大安區、松山區、中正區、大同區、萬華區、內湖區、文山區、北投區、士林區、南港區
- **手動切換機制**：CURRENT_CENTER_INDEX = 0-11選擇不同區域
- **簡單操作**：只需修改一個數字即可切換搜尋區域
- **完全避免重複**：每個區域的搜尋範圍不重疊

#### 4. 搜尋記錄和斷點續傳功能 ⭐ 系統優化
**記錄機制：**
- **JSON檔案**：search_log.json記錄搜尋進度
- **組合鍵值**：query_key = f"{query}_{GRID_SIZE}_{GRID_RADIUS}"
- **自動過濾**：載入已完成的網格點，只搜尋剩餘的點
- **進度顯示**：顯示已完成、剩餘網格點數量

**斷點續傳：**
- **自動恢復**：程式重啟後自動從上次位置繼續
- **安全保存**：每10個網格點自動保存進度
- **錯誤處理**：即使某個網格點失敗也會保存進度

### 🔧 技術突破點

#### Google Places API限制的完全突破
- **從60筆到數千筆**：網格搜尋可獲得數千筆餐廳資料
- **覆蓋全台北**：12個區域完全覆蓋台北市
- **智能去重複**：避免同一餐廳被重複收集
- **高效搜尋**：每個區域最多7,260筆（121×60），去重複後約1,000-3,000筆

#### 搜尋系統的智能化
- **自動記錄**：搜尋進度自動記錄和恢復
- **區域切換**：簡單的數字調整即可切換搜尋區域
- **進度追蹤**：詳細的搜尋進度顯示和統計
