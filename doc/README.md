# 🍷 BYOB 專案開發文檔

## 📋 專案概述

BYOB (Bring Your Own Bottle) 是一個餐廳資訊平台，讓消費者可以找到支援自帶酒水的餐廳。本專案使用 WordPress + WooCommerce 作為後端，並整合自定義的餐廳業者會員系統。

### 🎯 專案目標
- 建立完整的餐廳業者會員系統
- 提供餐廳資料管理功能
- 實現自動化餐廳資料收集
- 建立專業的餐廳邀約追蹤機制
- 建立 SEO 優化與內容行銷系統
- 整合SendGrid Email發送系統
- 建立餐廳業者註冊通知系統
- 建立專業內容行銷策略

## 📅 2025年9月18日 - Google Places API爬蟲程式60筆限制突破 ⭐ 重大技術突破

### 🎯 今日主要目標
突破Google Places API的60筆限制，建立網格搜尋系統，大幅提升餐廳資料收集能力。

### ✅ 已完成工作

#### 1. Google Places API限制分析 ⭐ 技術分析
**問題發現：**
- **API硬性限制**：Google Places API Text Search最多只能返回60筆結果（3頁×20筆）
- **無法突破**：這是Google的API限制，無法透過增加MAX_RESULTS來解決
- **搜尋範圍限制**：單一搜尋關鍵字無法獲得更多資料

**解決方案設計：**
- **網格搜尋策略**：將搜尋區域劃分為多個小網格
- **多中心點搜尋**：使用不同區域作為搜尋中心點
- **搜尋記錄機制**：避免重複搜尋相同區域

#### 2. 網格搜尋系統實作 ⭐ 核心功能
**技術實作：**
- **網格點生成**：每個中心點生成11×11=121個網格點
- **搜尋半徑設定**：0.03度（約3公里）的搜尋範圍
- **位置參數**：每個網格點使用location參數提升搜尋精確度
- **去重複機制**：使用place_id避免同一餐廳被重複收集

**系統架構：**
- **多中心點搜尋**：12個台北主要區域作為搜尋中心點
- **搜尋記錄檔案**：search_log.json記錄已搜尋的網格點
- **斷點續傳**：支援中斷後從上次位置繼續搜尋
- **進度保存**：每10個網格點自動保存進度

#### 3. 多中心點搜尋系統建立 ⭐ 系統架構
**中心點設定：**
- **12個台北區域**：信義區、中山區、大安區、松山區、中正區、大同區、萬華區、內湖區、文山區、北投區、士林區、南港區
- **手動切換機制**：CURRENT_CENTER_INDEX = 0-11選擇不同區域
- **簡單操作**：只需修改一個數字即可切換搜尋區域
- **完全避免重複**：每個區域的搜尋範圍不重疊

#### 4. 搜尋記錄和斷點續傳功能 ⭐ 系統優化
**記錄機制：**
- **JSON檔案**：search_log.json記錄搜尋進度
